c++11互斥体

C++11引入了许多新特性来支持多线程编程，其中包括互斥体（mutex）和其他同步机制。互斥体提供了一种机制，用于确保多个线程在同一时刻不会访问共享资源 ，从而避免数据竞争。下面是对C++11互斥体的详细介绍。

1. std::mutex

std::mutex 是最基本的互斥体类型，用于保护共享数据，以防止多个线程同时访问。互斥体可以显式地加锁和解锁。

基本用法

#include <iostream>
#include <thread>
#include <mutex>
using namespace std;

mutex mtx;

void print_thread_id(int id) {
    mtx.lock();
    cout << "Thread " << id << std::endl;
    mtx.unlock();
}

int main() {
    thread t1(print_thread_id, 1);
    thread t2(print_thread_id, 2);

    t1.join();
    t2.join();

    return 0;
}

在这个例子中，mtx.lock() 和 mtx.unlock() 确保了只有一个线程能够在同一时刻访问共享资源。

2. std::lock_guard

std::lock_guard 是一种方便的 RAII 机制，用于自动管理互斥体的锁定和解锁。它在构造时锁定互斥体，在析构时解锁互斥体，确保互斥体在作用域结束时总是被解锁。

基本用法

#include <iostream>
#include <thread>
#include <mutex>

std::mutex mtx;

void print_thread_id(int id) {
    std::lock_guard<std::mutex> lock(mtx);
    std::cout << "Thread " << id << std::endl;
}

int main() {
    std::thread t1(print_thread_id, 1);
    std::thread t2(print_thread_id, 2);

    t1.join();
    t2.join();

    return 0;
}

在这个例子中，std::lock_guard 确保了 mtx 在函数 print_thread_id 中的整个作用域内都是锁定的。

3. std::unique_lock

std::unique_lock 是一种更灵活的互斥体锁管理器。与 std::lock_guard 相比，std::unique_lock 支持延迟锁定、锁定超时和锁的解锁/重新锁定。

基本用法

#include <iostream>
#include <thread>
#include <mutex>

std::mutex mtx;

void print_thread_id(int id) {
    std::unique_lock<std::mutex> lock(mtx);
    std::cout << "Thread " << id << std::endl;
}

int main() {
    std::thread t1(print_thread_id, 1);
    std::thread t2(print_thread_id, 2);

    t1.join();
    t2.join();

    return 0;
}

延迟锁定和手动锁定/解锁

#include <iostream>
#include <thread>
#include <mutex>

std::mutex mtx;

void print_thread_id(int id) {
    std::unique_lock<std::mutex> lock(mtx, std::defer_lock); // 延迟锁定
    // 其他操作
    lock.lock(); // 手动锁定
    std::cout << "Thread " << id << std::endl;
    lock.unlock(); // 手动解锁
    // 其他操作
}

int main() {
    std::thread t1(print_thread_id, 1);
    std::thread t2(print_thread_id, 2);

    t1.join();
    t2.join();

    return 0;
}

带超时的锁定

#include <iostream>
#include <thread>
#include <mutex>
#include <chrono>

std::mutex mtx;

void try_lock_for_example() {
    std::unique_lock<std::mutex> lock(mtx, std::try_to_lock);
    if (lock.owns_lock()) {
        std::cout << "Lock acquired." << std::endl;
    } else {
        std::cout << "Lock not acquired." << std::endl;
    }
}

int main() {
    std::thread t1(try_lock_for_example);
    std::thread t2(try_lock_for_example);

    t1.join();
    t2.join();

    return 0;
}

4. std::timed_mutex 和 std::recursive_mutex

std::timed_mutex

std::timed_mutex 支持带超时的锁定操作。

#include <iostream>
#include <thread>
#include <mutex>
#include <chrono>

std::timed_mutex tmtx;

void try_lock_for_example() {
    if (tmtx.try_lock_for(std::chrono::seconds(1))) {
        std::cout << "Lock acquired." << std::endl;
        tmtx.unlock();
    } else {
        std::cout << "Lock not acquired within timeout." << std::endl;
    }
}

int main() {
    std::thread t1(try_lock_for_example);
    std::thread t2(try_lock_for_example);

    t1.join();
    t2.join();

    return 0;
}

std::recursive_mutex

std::recursive_mutex 允许同一个线程多次锁定同一个互斥体，而不会死锁。

#include <iostream>
#include <thread>
#include <mutex>

std::recursive_mutex rmtx;

void recursive_function(int count) {
    if (count <= 0) return;
    rmtx.lock();
    std::cout << "Count: " << count << std::endl;
    recursive_function(--count);
    rmtx.unlock();
}

int main() {
    std::thread t1(recursive_function, 10);
    t1.join();

    return 0;
}

总结

C++11 提供了多种类型的互斥体和相应的锁管理工具来确保线程安全：

  • std::mutex：基本的互斥体类型。
  • std::lock_guard：RAII 风格的互斥体管理。
  • std::unique_lock：更灵活的锁管理，支持延迟锁定、超时锁定等。
  • std::timed_mutex：支持带超时的锁定操作。
  • std::recursive_mutex：允许同一线程多次锁定。

这些工具帮助程序员更方便和安全地编写多线程程序，确保对共享资源的访问是互斥的，从而避免数据竞争和潜在的并发问题。
